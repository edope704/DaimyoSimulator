# DaimyoSimulator — System Test Document

## 1. Purpose

This document records the validation of DaimyoSimulator acceptance criteria. It follows the university requirement that each acceptance criterion should be checked with an OK/KO result, comments, and date.

The focus is system-level validation: the complete application should be executed and observed through the available UI, logs, and saved files. Unit test results are reported separately through the JUnit report generated by Maven.

## 5. Manual execution checklist

1. Run `mvn clean test`.
2. Launch the desktop application.
3. Create a new village.
4. Verify the map, HUD, and initial resources.
5. Test camera pan and zoom.
6. Place valid and invalid buildings.
7. Select buildings and inspect panels.
8. Press next tick and compare HUD updates.
9. Test pause and speed controls if automatic playback is implemented.
10. Activate policies and verify active/cooldown display.
11. Save and load the village.
12. Fill the OK/KO, date, tester, and comments columns.

---

## 6. Notes

- A KO result should include the observed behavior and the expected behavior.
- A fixed random seed is recommended during system tests that depend on random events.
- UI validation should focus on behavior and architecture boundaries, not on final art quality.
- Final art assets are created manually by the team and are not generated by tests.
